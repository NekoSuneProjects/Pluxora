# Pluxora Plugin Publishing Terms

These terms apply to repositories, plugin packages, and releases that use the
`pluxora-package` GitHub topic, are submitted for Pluxora discovery, or are
otherwise presented as Pluxora plugins.

By publishing a plugin under the `pluxora-package` topic, you agree to these
terms.

## Plugin Safety Rules

Plugins must not contain, install, download, or execute:

- malware, spyware, ransomware, destructive code, or persistence mechanisms;
- token loggers, credential stealers, session stealers, or phishing flows;
- IP grabbers, doxxing tools, unauthorized tracking, or hidden data collection;
- backdoors, hidden remote command execution, or undisclosed update loaders;
- Discord raid, spam, harassment, scam, exploit, or abuse tooling;
- code intended to bypass Discord, GitHub, Pluxora, or platform safety systems;
- dangerous dependency install scripts that are not clearly disclosed;
- illegal content or code intended to help illegal activity.

## Required Plugin Conduct

Plugin publishers must:

- own the code they publish or have permission to publish it;
- respect all dependency, community plugin, asset, and third-party licenses;
- include a clear license for their own plugin;
- accurately describe what the plugin does;
- request only the permissions the plugin actually needs;
- disclose external services, network calls, telemetry, analytics, storage, or
  data collection;
- avoid collecting secrets, tokens, IP addresses, personal data, or guild data
  unless the collection is necessary, visible, and lawful;
- follow Discord's Terms of Service and all applicable laws.

## Enforcement

If a plugin appears to be malicious, abusive, unsafe, illegal, misleading, or
in violation of these terms, Pluxora maintainers may take action. Actions may
include:

- removing or blocking the plugin from Pluxora discovery;
- warning users or maintainers;
- reporting the repository or package to GitHub, Discord, hosting providers,
  security vendors, or other relevant parties;
- publishing security guidance for affected users;
- refusing support for affected plugins or publishers.

Reports from users, maintainers, automated scans, or security researchers may
be used to decide whether action is needed. Pluxora maintainers may act before
contacting the plugin publisher when user safety is at risk.

## Community Plugin Rights

Community plugins keep their own ownership, copyright, and license terms.
Users and publishers must follow each plugin's license. These terms do not
transfer plugin ownership to Pluxora.

## Reporting Unsafe Plugins

If you find a plugin that is malicious, abusive, unsafe, collecting data
without consent, grabbing IPs, stealing tokens, or doing anything dangerous,
contact us on Discord:

https://nekosunevr.co.uk/?redirect=discord

Include the plugin name, repository link, what happened, and any logs or proof
you can safely share.

## No Guarantee

Pluxora discovery does not mean a plugin has been audited, approved, or proven
safe. Users should review plugin code and dependencies before installing them.
